## 完成时间

2026-03-11 16:00

## 完成内容

### 1. 修改 `downsample_corridor.py`
- 抽出 `run_downsample()` 函数，接受纯 Python 参数而非 argparse Namespace
- 移除 `input()` 交互确认，改为 `raise FileExistsError` + `--force`
- `main()` 仅做 argparse → 调用 `run_downsample()`

### 2. 修改 `split_corridor_runs.py`
- 抽出 `run_split()` 函数，接受纯 Python 参数
- 移除 `input()` 交互确认，改为 `raise FileExistsError` + `--force`
- `main()` 仅做 argparse → 调用 `run_split()`

### 3. `derive_stage1_datasets.py`
- 已有 `main()` 函数，接受 sys.argv 控制，无 `input()`，无需修改

### 4. 修改 `batch_export.py`
- `--split` 参数 help 标记为 `[已弃用]`
- 推荐使用 `split_corridor_runs.py` 或 `corridor_dataset_pipeline.py`

### 5. 新建 `scripts/corridor_dataset_pipeline.py`
- 统一入口，支持 4 种模式：
  - `export_only`: 仅导出
  - `prepare`: export + downsample + split
  - `derive_only`: 仅派生
  - `all`: 全流程
- 支持 `--skip_export`, `--skip_downsample`, `--skip_split`, `--skip_derive`
- 各阶段调用已有脚本的 `run_*()` 函数或 `main()`
- 自动计时，生成 `pipeline_log.json`

## 用法示例

```bash
# 全流程 (有 rosbag)
python scripts/corridor_dataset_pipeline.py --mode all --bag_dir ./data/bags --force

# 从已降采样数据开始 (跳过 export 和 downsample)
python scripts/corridor_dataset_pipeline.py --mode all --skip_export --skip_downsample --force

# 仅派生任务数据集
python scripts/corridor_dataset_pipeline.py --mode derive_only --force

# 单独运行某阶段 (仍可用原脚本)
python scripts/downsample_corridor.py --src_root ./data/corridor_all --dst_root ./data/corridor_balanced --force
python scripts/split_corridor_runs.py --src_root ./data/corridor_balanced --dst_root ./data/corridor --force
python scripts/derive_stage1_datasets.py --src_root ./data/corridor --dst_root ./data/stage1 --task all --force
```
