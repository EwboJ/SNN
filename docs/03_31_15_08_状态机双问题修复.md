# 状态机双问题修复进展 (2026-03-31 15:08)

## 修改文件
- `controllers/hierarchical_state_machine.py`

## 解决的两个系统问题

### 问题 1: TURN -> RECOVER 过度依赖 timeout
**原因**：之前 TURN 退出仅有两条路径——recover_support 累计确认 或 timeout 兜底，缺少中间退出条件。

**修复方案**：
1. 新增 `min_turn_steps` 门控：TURN 最少持续步数，过早不允许退出
2. 满足最小步数后，**三条优先级退出路径**：
   - **优先级 1**：`recover_signal_confirmed` — recover_support_count 达标
   - **优先级 2**：`recover_by_low_turn_and_low_omega` — turn_votes ≤ 阈值 且 |omega_raw| ≤ 阈值 持续足够步数
   - **优先级 3**：`turn_timeout` — max_turn_steps 强制兜底（仅在前两个条件都不满足时触发）
3. recover_support_count 使用渐减模式（`max(0, count-1)`）而非立即清零，基于 recover_votes >= majority_thr 判断

### 问题 2: Junction 方向在 APPROACH 早期过早锁存
**原因**：进入 APPROACH 后立即允许 junction 锁存，可能在真正接近路口之前就把方向锁错。

**修复方案**：
1. 新增 `min_approach_steps_before_junction_lock`：步数不足时延迟锁存
2. 新增 `min_turn_votes_before_junction_lock`：Turn 信号太弱时延迟锁存
3. 锁存允许条件：步数达标 **且** (turn_votes 达标 **或** Turn 信号上升趋势)
4. 延迟期间继续累积 junction votes，但不正式锁存为 locked_turn_dir

## 新增构造函数参数及默认值

|------|--------|------|
| `min_turn_steps` | 5 | TURN 最小步数，之前不允许退出 |

| `straight_recover_omega_thresh` | 0.3 | |omega| 低于此值视为可安全切回直行 |
| `straight_recover_hold_steps` | 3 | 低 turn + 低 omega 需持续的步数 |
| `min_approach_steps_before_junction_lock` | 3 | APPROACH 中 junction 最早锁存步数 |
| `min_turn_votes_before_junction_lock` | 2 | junction 锁存需要的最低 turn 票数 |

## Transition Reason 区分

- `recover_signal_confirmed` — Recover 信号累计确认退出
- `recover_by_low_turn_and_low_omega` — 低 turn 票数 + 低 omega 持续退出
- `turn_timeout` — 超时兜底退出
- `junction_lock_deferred_by_early_approach` — APPROACH 步数不足延迟锁存
- `junction_lock_deferred_by_weak_turn_signal` — Turn 信号太弱延迟锁存
