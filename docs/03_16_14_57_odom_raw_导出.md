# 进展：corridor_export.py 支持 odom_raw

## 完成时间

2026-03-16 15:03

## 完成内容

### 1. 修改 `corridor_export.py`

新增 `/odom_raw` 导出支持，不破坏现有 labels.csv 和 meta.json 兼容性。

**新增/修改：**
- `--odom-topic` 参数，默认 `/odom_raw`
- `DEFAULT_CONFIG["bag"]` 新增 `odom_topic` 字段
- `read_bag_messages()` 返回三元组 `(image_msgs, cmd_vel_msgs, odom_msgs)`
- `_extract_odom()` 从 Odometry 消息提取 `(ts, x, y, yaw, linear_v, angular_w)`
  - 支持 `nav_msgs/Odometry` 和 `geometry_msgs/PoseStamped`
  - 四元数→yaw 转换
- `write_odom_csv()` 输出 `odom_raw.csv`
- `write_meta_json()` 新增 `odom_available`、`odom_topic`、`odom_count`

**odom_raw.csv 字段：**
```
timestamp_ns, x, y, yaw, linear_v, angular_w
```

**兼容性：**
- labels.csv 字段不变
- 若 bag 中无 odom topic，仅 warning，不报错，跳过 odom_raw.csv

### 2. 修改 `scripts/batch_export.py`

- 新增 `--odom_topic` 参数
- 通过 cfg 透传给 `run_export()`

### 3. 修改 `scripts/corridor_dataset_pipeline.py`

- Export 参数区新增 `--odom_topic`
- `stage_export` 中透传给 `batch_export` 的 `export_argv`

## 用法

```bash
# 单个 bag 导出 (默认含 odom_raw)
python corridor_export.py --bag ./nav --output ./data/run1

# 指定 odom topic
python corridor_export.py --bag ./nav --output ./data/run1 --odom-topic /odom

# 批量导出
python scripts/batch_export.py --bag_dir ./bags --output_dir ./data/all --odom_topic /odom_raw

# pipeline 全流程
python scripts/corridor_dataset_pipeline.py --task_type straight_keep --mode all --odom_topic /odom_raw
```
